# vsForms Production Deployment Checklist

This document serves as a comprehensive guide for deploying vsForms to a production environment. It covers prerequisites, configuration steps, deployment procedures, and maintenance tasks to ensure a successful production setup.

## 1. Prerequisites

### Hardware Requirements
- **Server**: 
  - Minimum: 2 CPU cores, 4GB RAM, 20GB storage
  - Recommended: 4 CPU cores, 8GB RAM, 40GB+ storage
- **Network**: Reliable internet connection with outbound access

### Software Requirements
- **Docker**: Docker Engine 20.10+ and Docker Compose v2+
- **Git**: To clone the repository
- **Domain**: A registered domain name for your vsForms instance
- **SMTP Server**: For sending emails (notifications, password resets, etc.)

## 2. Pre-Deployment Tasks

### Domain Setup
- [ ] Register a domain or set up a subdomain for vsForms
- [ ] Configure DNS records to point to your server's IP address
- [ ] (Optional) Set up SSL certificates (Let's Encrypt recommended)

### Server Preparation
- [ ] Ensure all ports are properly configured:
  - Port 80/443 (HTTP/HTTPS) open for web access
  - SSH port secured for remote administration
- [ ] Install Docker and Docker Compose
- [ ] Configure firewall rules to allow required traffic
- [ ] Set up a regular backup strategy

## 3. Environment Configuration

### Clone the Repository
```bash
git clone https://github.com/JhumanJ/vsForms.git
cd vsForms
```

### Configure Environment Variables

#### API Environment (api/.env)
- [ ] Set `APP_URL` to your production domain
- [ ] Configure `APP_ENV=production`
- [ ] Set `FRONT_URL` to your frontend URL (same as APP_URL for most setups)
- [ ] Configure database settings (PostgreSQL recommended):
  ```
  DB_CONNECTION=pgsql
  DB_HOST=db
  DB_PORT=5432
  DB_DATABASE=forge
  DB_USERNAME=forge
  DB_PASSWORD=strong-password-here
  ```
- [ ] Configure email settings:
  ```
  MAIL_MAILER=smtp
  MAIL_HOST=your-smtp-server.com
  MAIL_PORT=587
  MAIL_USERNAME=your-username
  MAIL_PASSWORD=your-password
  MAIL_ENCRYPTION=tls
  MAIL_FROM_ADDRESS=noreply@yourdomain.com
  MAIL_FROM_NAME="Your Company Name"
  ```
- [ ] Set up file storage (S3 recommended for production):
  ```
  FILESYSTEM_DISK=s3
  AWS_ACCESS_KEY_ID=your-access-key
  AWS_SECRET_ACCESS_KEY=your-secret-key
  AWS_DEFAULT_REGION=your-region
  AWS_BUCKET=your-bucket-name
  AWS_URL=https://your-bucket-name.s3.amazonaws.com
  ```
- [ ] Configure Redis (used for cache, sessions, and queues):
  ```
  REDIS_HOST=redis
  REDIS_PASSWORD=null
  REDIS_PORT=6379
  CACHE_DRIVER=redis
  SESSION_DRIVER=redis
  QUEUE_CONNECTION=redis
  ```
- [ ] Set up security keys (will be automatically generated by the setup script):
  ```
  APP_KEY=base64:random-key
  JWT_SECRET=random-secret
  FRONT_API_SECRET=shared-secret-with-frontend
  ```

#### Client Environment (client/.env)
- [ ] Set `NUXT_PUBLIC_APP_URL` to your production domain
- [ ] Configure `NUXT_PUBLIC_API_BASE` to your API URL (typically `/api` if using the included Nginx proxy)
- [ ] Set `NUXT_PUBLIC_ENV=production`
- [ ] Configure `NUXT_API_SECRET` to match the `FRONT_API_SECRET` in the API environment
- [ ] (Optional) Set up analytics and monitoring:
  ```
  NUXT_PUBLIC_GOOGLE_ANALYTICS_CODE=your-ga-code
  NUXT_PUBLIC_AMPLITUDE_CODE=your-amplitude-code
  NUXT_PUBLIC_CRISP_WEBSITE_ID=your-crisp-id
  ```

## 4. Docker Deployment

### Using the Setup Script (Recommended)
```bash
chmod +x scripts/docker-setup.sh
./scripts/docker-setup.sh
```

### Manual Deployment
```bash
# Set up environment files if not using the script
cp api/.env.docker api/.env
cp client/.env.docker client/.env
# Edit the files with your production settings

# Start the containers
docker compose up -d
```

### Post-Deployment Verification
- [ ] Check if all containers are running:
  ```bash
  docker compose ps
  ```
- [ ] Verify logs for any errors:
  ```bash
  docker compose logs -f
  ```
- [ ] Access your vsForms instance at your domain
- [ ] Log in with the default credentials:
  - Email: `admin@vsForms.com`
  - Password: `password`
- [ ] Change the default admin email and password immediately
- [ ] Test form creation and submission functionality
- [ ] Verify email notifications are working

## 5. Advanced Configuration

### SSL/HTTPS Setup
- [ ] Set up an SSL certificate using Let's Encrypt or another provider
- [ ] Configure Nginx to use SSL:
  ```bash
  # Create docker-compose.override.yml
  cat > docker-compose.override.yml << EOL
  services:
    ingress:
      volumes:
        - ./nginx.ssl.conf:/etc/nginx/templates/default.conf.template
        - ./ssl:/etc/nginx/ssl
      ports:
        - "443:443"
  EOL
  ```

### Database Optimization
- [ ] Configure PostgreSQL for better performance:
  ```bash
  # Create docker-compose.override.yml or add to existing
  cat > docker-compose.override.yml << EOL
  services:
    db:
      command: postgres -c shared_buffers=256MB -c work_mem=8MB
      volumes:
        - ./postgresql.conf:/etc/postgresql/postgresql.conf
  EOL
  ```

### Storage Configuration
- [ ] For S3 storage, set up proper CORS configuration:
  ```json
  [
    {
      "AllowedHeaders": ["*"],
      "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
      "AllowedOrigins": ["https://your-domain.com"],
      "ExposeHeaders": []
    }
  ]
  ```

## 6. Security Considerations

### General Security
- [ ] Use strong, unique passwords for database and admin accounts
- [ ] Enable CAPTCHA protection for public forms:
  ```
  NUXT_PUBLIC_H_CAPTCHA_SITE_KEY=your-hcaptcha-key
  HCAPTCHA_SECRET=your-hcaptcha-secret
  ```
  or
  ```
  NUXT_PUBLIC_RE_CAPTCHA_SITE_KEY=your-recaptcha-key
  RECAPTCHA_SECRET=your-recaptcha-secret
  ```
- [ ] Implement a Web Application Firewall (WAF) for additional protection
- [ ] Set up rate limiting to prevent abuse

### Docker Security
- [ ] Use non-root users in containers (default in vsForms images)
- [ ] Regularly update Docker images:
  ```bash
  docker compose pull
  docker compose up -d
  ```
- [ ] Implement Docker network isolation if needed

## 7. Backup Strategy

### Database Backups
- [ ] Set up automated PostgreSQL backups:
  ```bash
  # Add to crontab
  0 2 * * * docker compose exec -T db pg_dump -U forge forge > /path/to/backups/vsForms_$(date +\%Y\%m\%d).sql
  ```

### File Backups
- [ ] If using local storage, back up the storage directory:
  ```bash
  # Add to crontab
  0 3 * * * tar -czf /path/to/backups/vsForms_storage_$(date +\%Y\%m\%d).tar.gz /path/to/vsForms/api/storage
  ```
- [ ] For S3 storage, set up versioning and lifecycle policies in your S3 bucket

### Configuration Backups
- [ ] Back up your environment files and custom configurations:
  ```bash
  # Add to crontab
  0 4 * * * tar -czf /path/to/backups/vsForms_config_$(date +\%Y\%m\%d).tar.gz /path/to/vsForms/api/.env /path/to/vsForms/client/.env /path/to/vsForms/docker-compose.override.yml
  ```

## 8. Maintenance Procedures

### Regular Updates
- [ ] Pull the latest changes:
  ```bash
  git pull origin main
  ```
- [ ] Update Docker images:
  ```bash
  docker compose pull
  docker compose up -d
  ```

### Troubleshooting Common Issues

#### Container Issues
```bash
# View detailed logs
docker compose logs -f

# Recreate containers
docker compose down
docker compose up -d
```

#### Database Issues
```bash
# Check database status
docker compose exec db pg_isready

# View database logs
docker compose logs db
```

#### Cache Issues
```bash
# Clear application cache
docker compose exec api php artisan cache:clear

# Clear config cache
docker compose exec api php artisan config:clear

# Clear route cache
docker compose exec api php artisan route:clear
```

#### Permission Issues
```bash
# Fix storage permissions
docker compose exec api chmod -R 775 storage
docker compose exec api chown -R www-data:www-data storage
```

## 9. Monitoring and Logging

### Container Monitoring
- [ ] Set up Docker container monitoring:
  ```bash
  # Install ctop for container monitoring
  sudo wget https://github.com/bcicen/ctop/releases/download/v0.7.7/ctop-0.7.7-linux-amd64 -O /usr/local/bin/ctop
  sudo chmod +x /usr/local/bin/ctop
  
  # Run ctop to monitor containers
  ctop
  ```

### Log Management
- [ ] Set up log rotation:
  ```bash
  # Create docker-compose.override.yml or add to existing
  cat >> docker-compose.override.yml << EOL
  services:
    api:
      logging:
        driver: "json-file"
        options:
          max-size: "10m"
          max-file: "3"
  EOL
  ```
- [ ] Consider using a log aggregation service for centralized logging

## 10. Scaling Considerations (Optional)

### Horizontal Scaling
- [ ] Configure a load balancer for multiple API instances
- [ ] Separate the database to a dedicated server or managed service
- [ ] Set up Redis clustering for larger deployments

### Vertical Scaling
- [ ] Increase container resources:
  ```bash
  # Create docker-compose.override.yml or add to existing
  cat >> docker-compose.override.yml << EOL
  services:
    api:
      deploy:
        resources:
          limits:
            cpus: '2'
            memory: 2G
  EOL
  ```
